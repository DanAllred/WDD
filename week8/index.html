With the transform property you can translate, rotate, scale and/or skey any element
with all the above mentioned functions, note you need transform: functionname() for it to work
The translation function lets you move elements similar to the function position: relative
Note that the translated element can only be moved relative to its current position

translate(x,y) moves the function along the x and y coords. example is transform: translate(45px, -45px)
to move in one direction only use transform: translatex() or translatey()
Also note Transforms require vendor prefixing for IE9, Android up to 4.4.3, iOS8, and
Blackberry 10. To make the aforementioned code work in IE9 and older mobile
WebKit browsers, you would include the following:

-webkit-transform: translate(45px,-45px); /* iOS8, Android
➥4.4.3, BB10 */
-ms-transform: translate(45px,-45px); /* IE9 only */
transform: translate(45px,-45px);

final note on translate is that they don't work on inline elements so don't forget display: inline-block

---------------------------------------------------------------------------------------------------

the scale() function works with the same x and y axis but to scale an element. in otherwords
make it bigger or smaller. scale(2) would double, scale(.5) would halve it, where scale(1) would maintain.
note that scaled elementes shrink or grow towards or from the center.

---------------------------------------------------------------------------------------------------

rotate() spins an element around the center by degree. positive degrees rotate clockwise, negative, counterclockwise
example: transform: rotate(10deg)

---------------------------------------------------------------------------------------------------

skew() can skew an element along x and y axis by degrees. if only one paramter is used then only x axis will be effected.
skew(15deg, 4deg)

using transform-origin: allows you to change the point of origin of an element by percentage
transform-origin: 10% 10%

The order of transform functions does matter: if you rotate before translating,
your translate direction will be on the rotated axis. The rightmost square in Figure
8.8 was translated then rotated with transform: translateX(200px)
rotate(135deg);. The leftmost square was rotated first then translated along
the newly rotated axis: transform: rotate(135deg) translateX(200px);.

You can use transitions which allow values of CSS properties to change over time creating animations.

1. Declare the original state of the element in the default style declaration.
2. Declare the final state of your transitioned element; for example, a :hover state.
3. Include the transition functions in your default style declaration using the
transition properties, including: transition-property, transition-duration,
transition-timing-function, and transition-delay.

CSS example

.li {
 transition-property: transform;
 transition-duration: 0.1s; #duration is in seconds
 transition-timing-function: ease-out; #https://developer.mozilla.org/en-US/docs/Web/CSS/transition-timing-function
 trnasition-delay: 50ms; #ms = milliseconds
}

you can have multiple transitions seperated by a , and a space. order is important as each transition corresponds to it's position

CSS animations control the steps of an animation via keyframes
Keyframes use from and to defined with a percentage which determines
how far long the animation each keyframe is located. This means that the order they apear
does not matter, it is all about the percentage

@keyframes moveRight {
 from {
  transform: translateX(-50%);
 }
 to {
  transform: translateX(50%);
 }
}
@keyframes appearDisappear {
 0%, 100% {
  opacity: 0;
 }
 20%, 80% {
  opacity: 1;
 }
}
@keyframes bgMove {
 100% {
  background-position: 120% 0;
 }
}

The above example will cause the element to start invisible, fade into visible at 20%
duration, then at 80% fade out again.

steps( n, jumpterm) Displays the transition along n stops along the transition, displaying each stop for equal lengths of time.
For example, if n is 5, there are 5 steps. Whether the transition holds temporarily at 0%, 20%, 40%, 60% and 80%, on the
20%, 40%, 60%, 80% and 100%, or makes 5 stops between the 0% and 100% along the transition, or makes 5 stops including the 0%
and 100% marks (on the 0%, 25%, 50%, 75%, and 100%) depends on which jump term is used
for jumpterms https://developer.mozilla.org/en-US/docs/Web/CSS/transition-timing-function ---------------------------------------------------------------------------------------------------

animations and transitions are great because they provide a lighter animation than cpu hogger JavaScript

The Canvas API provides a means for drawing graphics via JavaScript and the HTML <canvas> element.
Among other things, it can be used for animation, game graphics, data visualization, photo manipulation,
and real-time video processing. example:

this goes in html:
<canvas id="canvas"></canvas>

in javascript:
const canvas = document.getElementById('canvas');
const ctx = canvas.getContext('2d');

ctx.fillStyle = 'green';
ctx.fillRect(10, 10, 150, 100);

Result would be a green rectangle. Note that height and width of the canvas is declared directly in the html element
To make a gradient that starts as blue and begins to blend into white halfway down the gradient, we can specify a blue
color stop with an offset of 0 and a purple color stop with an offset of 1:

gradient.addColorStop(0, "blue");
gradient.addColorStop(1, "white");

Canvas API can also be used for altering videos. Things like green screens and visual effects added to videos.

Once the draw function has been written for a frame we can then call that same function for each frame we want to manipulate.

SVG stands for Scalable Vector Graphics which uses XML. Vector images maintain quality as you blow them up or make them smaller
Many of the same things done in canvas can be done in SVG. Lots of open source ools can be used as well. Also more accessible than canvas.

<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 400 400">
 <circle cx="50" cy="50" r="25" fill="red"/>
</svg>

viewBox attribute defines the starting location, width, and height. Circle element defines the a circle, cx, cy, X and Y coords of the
center of the circle. r=radius and fill is fill style.

or you can use http://inkscape.org/ to help create SVG images.

Raphaël (http://raphaeljs.com/is) an open-source JavaScript library that makes drawing and animating with SVG much easier.

create a container by using the raphael method, then passing the html element name, width and height.
Now add image variable by calling on the conatiner. give the parameters of the path, starting coords(where the image goes), width and height of the image.

Drag and Drop API- think about how you add files to dropbox by dragging and dropping.

There are several steps to adding drag and drop to your page:
1. Set the draggable attribute on any HTML elements you’d like to be draggable. (draggable="true"
2. Add an event listener for the dragstart event on any draggable HTML elements.
3. Add an event listener for the dragover and drop events on any elements that you want to have accept dropped items.

DataTransfer lets us define two pieces of information:
■ the type of data we’re saving of the draggable element
■ the value of the data itself

dragover and drop events are monitored to do precisly what they say. JavaScript’s preventDefault method allows us to ensure that the default behavior
for an element does not happen. in the case of drag and drop we want to be able to drag something over another thing and drop it on it.
