Global vs local scope

Global scope means a variable declared outside of a block is accesible everywhere in the program. should keep
global variables to a minimum to avoid issues of common variable names overwriting eachother.

Local Scope means the variable is only accesible inside the block using it. *unless let or const were not used.

Array is an ordered list with square brackets - const myArray = [];

array.pop() removes last item. array.shift()removes first item
concat() method can be used to merge an array with one or more arrays:

avengers.concat(['Hulk','Hawkeye', 'Black Widow']);
<< ['Captain America', 'Iron Man', 'Thor', 'Hulk', 'Hawkeye', 'Black



splice() method removes items from an array then inserts new items in their place. For example, the following code
removes the string 'Hulk' and replaces it with 'Scarlett Witch':

avengers.splice(3, 1, 'Scarlet Witch');
<< ['Hulk']

This is a destructive operation as it changes the value of the array, as we can see below:

avengers
<< ['Captain America', 'Iron Man', 'Thor', 'Scarlet Witch', 'Hawkeye', 'Black widow']

note that you can still use the splice method to insert without removing... avengers.splice(3, 0, 'Dan Allred')

To access the values in a multidimensional array, we use two indices: one to refer to the item's place in the outer array,
and one to refer to its place in the inner array:

The spread operator can be used to flatten multi-dimensional arrays. Flattening an array involves removing all nested arrays
so all the values are on the same level in the array. You can see an example of a flattened array below:

const summer = ['Jun', 'Jul', 'Aug'];
const winter = ['Dec', 'Jan', 'Feb'];
const nested = [ summer, winter ];
<< [ [ 'Jun', 'Jul', 'Aug' ], [ 'Dec', 'Jan', 'Feb' ] ]

const flat = [...summer, ...winter];
<< [ 'Jun', 'Jul', 'Aug', 'Dec', 'Jan', 'Feb' ]

The switch operator can be used to make your code easier to follow when there are lots of conditions to test. The example above
can be rewritten using a switch statement like so:

switch (number) {
case 4:
console.log('You rolled a four');
break;
case 5:
console.log('You rolled a five');
break;
case 6:
console.log('You rolled a six');
break;
default:
console.log('You rolled a number less than four');
break;
}

The value you are comparing goes in parentheses after the switch operator. A case keyword is then used for each possible value that
can occur (4, 5, and 6 in the example above). After each case statement is the code that that needs to be run if that case occurs.

Parameters and arguments are terms that are often used interchangeably to represent values provided for the function
as an input. There is a subtle difference though: any parameters a function needs are set when the function is defined.
When a function is invoked, it is provided with arguments.

function square(x){
return x*x;
}
When we invoke this function, we need to provide an argument, which is the number to be squared:

we can also give default parameters:

function hello(name='World') {
console.log(`Hello ${name}!);
}

Callbacks

A function that is passed as an argument to another is known as a callback, or better yet, a function which is to be executed after another function
has finished execution. https://www.simplilearn.com/tutorials/javascript-tutorial/callback-function-in-javascript

function sing(song,callback) {
console.log(`I'm singing along to ${song}.`);
callback();
}

Can also use functions as callbacks in sort() method for arrays.
